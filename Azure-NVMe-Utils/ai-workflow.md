## Git Commit Rules

1. **Before any commit**, run `git diff --stat` to review changes, then show the exact proposed commit message inside a fenced `text` code block and ask for approval.
2. **After approval and commit**, ask separately whether to push.
3. When outputting **markdown content** for commits or pull requests, present it inside a fenced `text` code block.
